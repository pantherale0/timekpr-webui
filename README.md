# TimeKpr WebUI

A modern, web-based interface for managing TimeKpr-nExT parental controls across multiple computers in your network. 

This project uses a high-performance **Server-Client architecture**: the Flask-based Server hosts a real-time WebSocket hub, and target computers run a lightweight, standalone compiled **Rust Client Agent** (`timekpr-agent`) that connects outbound. No open ports, no SSH credentials, and robust security!

![Timekpr Dashboard](docs/dashboard.png)

---

## ⚡ Key Architectural Features

- **Outbound WebSockets**: Client agents connect *outbound* to the server WebSocket endpoint (`/ws`). Works natively behind NAT and firewalls with **no open inbound ports** (like SSH) on client machines!
- **🔐 HMAC-SHA256 Challenge-Response Auth**: The connection is secured with a cryptographically signed handshake challenge. The pre-shared key (`AGENT_TOKEN`) is **never** transmitted over the wire, rendering replay and eavesdropping attacks impossible.
- **🆔 Unique Host UUID Mapping**: Client machines are tracked using a persistent UUID auto-generated by the Rust agent. Dynamic IP (DHCP) changes will never break user mappings!
- **🌐 Real-time Online Indicators & Dynamic IP Snapshots**: The Admin panel displays active client connections with a real-time **green/grey Online/Offline indicator**. The client's current IP address is snapshotted dynamically in the database upon connection.
- **⏱️ Instant & Offline Commands**: Commands are executed instantly over persistent WebSockets. If a target computer is offline, the server automatically queues adjustments (weekly schedules, time extensions) and pushes them the second the client reconnects.
- **🦀 Compiled Rust Agent**: A standalone client daemon written in Rust. It runs asynchronously with tokio, auto-reconnects with exponential backoff, and has zero external runtime dependencies.
- **📊 Usage Tracking & Scheduling**: View usage statistics with interactive weekend-highlighted charts, adjust remaining time (+15m/-15m), and manage daily allowed intervals.

## 🚀 Getting Started

```mermaid

```

### Prerequisites
- **Docker and Docker Compose** installed on the server machine
- **Rust toolchain** (cargo/rustc) to compile the client agent
- **TimeKpr-nExT** installed on target client computers

---

### 1. Server Deployment (Docker)

1. **Clone the repository:**
   ```bash
   git clone https://github.com/adambie/timekpr-webui.git
   cd timekpr-webui
   ```

2. **Configure the Environment:**
   Open `docker-compose.yaml` and set a custom pre-shared key for `AGENT_TOKEN` and your timezone:
   ```yaml
   environment:
     - TZ=Europe/London
     - AGENT_TOKEN=your-custom-secure-token-here
   ```

3. **Start the Server:**
   ```bash
   docker-compose up -d --build
   ```
   The Flask server will start and listen on port `5000` (WebSocket endpoint available at `ws://YOUR_SERVER_IP:5000/ws`).

4. **Initialize Credentials:**
   Navigate to `http://localhost:5000` and log in with the default admin account:
   - **Username**: `admin`
   - **Password**: `admin`
   *⚠️ Go to the **Settings** page and change this default password immediately!*

---

### 2. Client Agent Deployment (Rust)

Perform these steps on **each client machine** running TimeKpr-nExT that you want to manage:

1. **Compile the Agent:**
   On a machine with Rust installed, compile the agent binary:
   ```bash
   cd agent
   cargo build --release
   ```
   The compiled standalone binary will be located at `agent/target/release/timekpr-agent`.

2. **Create the Configuration:**
   Copy the `timekpr-agent` binary to the client machine (e.g., to `/usr/local/bin/`).
   Create a configuration directory and file (requires root/sudo for primary path, or local directory fallback):
   Create `/etc/timekpr-agent/config.json`:
   ```json
   {
     "server_url": "ws://YOUR_SERVER_IP:5000/ws",
     "system_id": null,
     "agent_token": "your-custom-secure-token-here"
   }
   ```
   *Note: Set `agent_token` to match the `AGENT_TOKEN` configured in your server's `docker-compose.yaml`.*

3. **Launch the Agent:**
   Run the agent daemon (normally as root/sudo so it has permission to execute `timekpra`):
   ```bash
   sudo ./timekpr-agent
   ```
   Upon its very first launch, the agent detects that `system_id` is null, **automatically generates a persistent unique Host UUID**, saves it to `/etc/timekpr-agent/config.json`, and prints it:
   ```text
   ------------------------------------------------------------
   GENERATE NEW HOST UUID: 8f88c3a1-7ab3-4b68-b7eb-116b47cbf2fb
   PLEASE REGISTER THIS HOST UUID IN THE SERVER WEB UI PANEL!
   ------------------------------------------------------------
   ```
   *Copy this printed UUID; you will need it to map users on this machine.*

4. **Install as a Systemd Service (Optional but Recommended):**
   To keep the agent running in the background on boot, register it as a systemd service:
   Create `/etc/systemd/system/timekpr-agent.service`:
   ```ini
   [Unit]
   Description=Timekpr WebSocket Client Agent
   After=network.target

   [Service]
   Type=simple
   ExecStart=/usr/local/bin/timekpr-agent
   Restart=always
   RestartSec=5

   [Install]
   WantedBy=multi-user.target
   ```
   Enable and start the service:
   ```bash
   sudo systemctl daemon-reload
   sudo systemctl enable timekpr-agent.service
   sudo systemctl start timekpr-agent.service
   ```

---

### 3. Registering Users in Web UI

1. Open the WebUI and navigate to the **Admin Panel**.
2. Under **"Add New User"**, fill in:
   - **Username**: The Unix username on the client computer (e.g. `kid1`).
   - **System ID (Host UUID)**: The unique Host UUID printed by the client agent on startup (e.g. `8f88c3a1-7ab3-4b68-b7eb-116b47cbf2fb`).
3. Click **"Add User"**.
4. The server will perform a real-time HMAC authentication handshake over the agent's WebSocket connection, dynamically record its current IP snapshot, and mark the user as valid!

## 🔐 Advanced Security

- **Anti-Replay Handshake**: Standard WebSocket handshakes are vulnerable to packet interception. TimeKpr WebUI's server generates a cryptographic challenge (32-byte hex token) upon socket connection. The client combines the challenge and its `system_id` and signs it using HMAC-SHA256 with the pre-shared secret. Eavesdroppers intercepting the signature cannot replay it or retrieve the `AGENT_TOKEN`.
- **Minimized Attack Surface**: Client machines do not need to run SSH daemons or configure firewalls to accept port 22/80/443 traffic. The agent initiates outbound connections, so firewalls block all inbound traffic by default.
- **Privilege Separation**: The Rust client only responds to a hardcoded white-list of commands (`timekpra --userinfo`, `--settimeleft`, `--setalloweddays`, `--settimelimits`, `--setallowedhours`), preventing arbitrary command injection.

## 🔧 Troubleshooting

- **Agent remains "Offline" on Dashboard**:
  - Verify that the `agent_token` in `/etc/timekpr-agent/config.json` exactly matches the `AGENT_TOKEN` in the server's `docker-compose.yaml`.
  - Check that the `server_url` points to the correct server IP and port (`ws://SERVER_IP:5000/ws`).
  - Verify network connectivity between the client and server.
- **Command synching issues**:
  - Ensure the client agent is running as `root` (or via `sudo`), as standard Unix users do not have permissions to execute administrative `timekpra` modifications.
  - Review the agent daemon logs: `journalctl -u timekpr-agent.service`.
